General

Model > General tab allows you to configure the following settings:

  1. Export

  2. Writeback

#Export

In the Export settings, you will be able to configure the export functionality. There is also an option to include the navigation settings panel in the Export File (see Figure below).

#Writeback

ValQ custom visual provides the ability to write scenarios, planning series along with the simulation details to an URL which can collect the response and write it to a database, data warehouse, etc.

This enables planners to integrate ValQ's planning and what-if simulations with their business workflows.

  • Format- ValQ sends the data in a JSON format to the URL which can be processed easily.

  • Configurable - User can choose to export the entire model, or just the KPIs and simulated affected Nodes.


Figure: Writeback architecture

#How to configure Writeback in ValQ

  1. The Writeback settings are available under the Model > General > Writeback section.

Figure: Writeback settings

The following table lists out the fields:

FieldDescriptionOptions
URLURL of the backend data source to which ValQ will post the data
Button LabelLabel message displayed in the Writeback section
Button TextText message displayed on the Writeback button
Exporting NodesSelect the nodes to be exported
  • All Nodes
  • Only KPI and Key Inputs nodes
  • Only simulation affected nodes

Table: Writeback Properties

  1. This Writeback feature will now be available in the Simulate > Navigation Panel section on the runtime.

Note: The option is only enabled once the Writeback settings are configured in the Model section.

Figure: Writeback feature accessible from the Navigation Panel

  • Click the Writeback button
  • Select the Series and Scenarios that you want to export to the URL and submit.

Success: Your modeled scenarios and series have been successfully posted to the external URL.

#ValQ Writeback Schema

Writeback posts JSON to the writeback URL in a specific export schema.

This export format includes a JSON object with the following details:

  • Result array includes the Scenarios/Series fields and the respective Comments History

  • Simulated Nodes array includes fields details related to the change in values of the Simulated Nodes

  • Logged in ValQ User details

You can download the schema file here ValQ Writeback Schema.json

Figure: Writeback Schema format

#Writeback History

History of the Writeback logs is maintained and is accessed by clicking on the history icon right above the Writeback button in the Navigation panel.
