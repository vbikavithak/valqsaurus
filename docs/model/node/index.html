Node

In this section, you will learn everything related to node on ValQ's Model tab. In case you are new to node, it is recommended you visit Understanding Nodes in ValQ and then come back here to learn more about nodes.

Click on this icon on ValQ to access the Node options.

#Convert to Advanced Model

In our example, we have created a simple model and the below image is the screen displayed when you click on the Node option. Here, take a closer look to notice the nodes on the left panel are disabled. A simple model builds the model and configures the nodes as well.


To change the configuration of the nodes and to add/delete nodes you will need to use Convert Dynamic Model to Advanced Model option. This option is represented by the magic wand available next to the top node.


Note
The convert to advanced model option is irreversible.


The below image shows our example after it has been converted to an advanced model. Note that the child nodes have been activated now.

#2. Configuration

In Configuration, you will be able to configure the node details such as Node Type, Calculation Method, Time Aggregation, Desired Trend and Value Range with the values as shown in the below Figure.


Node Type: Node can be of two types - Normal and Constant. Normal node is the default type with all the options available while constant node sets the node value to a fixed figure.


Calculation Method: This option is available to choose a method of calculation for the node from the below-seen list of methods.


MethodDescription
1Data SourceGets the value directly from source data
2Sum ChildrenAdds all of the child node values
3Subtract ChildrenDeducts the other child node values from the first child node
4Multiply ChildrenMultiplies all of the child node values
5Divide ChildrenDivides the other child node values from the first child node
6FormulaCalculates and applies the custom formula entered by the user
7Linked to NodeThe value is linked to value from another node
8Template based on other nodeCaptures and applies the configuration from the other node
9ManualEnter the values manually

Note

On selecting 'Formula' and clicking on the formula field, you will be able to see a dropdown of the formulas that are supported. For a list of formulas that are available in ValQ, you can refer to the link Functions

The Normal node type presents the user with all the calculation methods as shown below.


The constant node type only makes the following calculation methods available for the user to choose from.


For the Calculation Method being selected as 'Data Source', you can follow the steps as explained in the following link: https://valq.com/blogs/3-methods-of-mapping-valq-nodes-to-data

For other options of Calculation Method, you can refer the link: https://valq.com/blogs/assigning-values-to-nodes-in-valq/


#Template Nodes

You can create a Template Node for an already existing node. Add a new node and select its Calculation Method as Template based on other nodes. Now select the node you want to copy under node to use as a template.

In the below Figure as an example, we create another node 'North' with a template the same as that of the 'East' node by following the steps mentioned above. Unique Id Prefix has also been set as North.


Here you can observe that 'North' node with the same structure as 'East' node has been created.


Source Key: The Source Key option gets displayed when the Calculation Method is selected as Data Source. Note that you shall select the Source Key that matches the node to get the correct value. The below image is an example that shows the Source Key with all the levels of category for quick identification and selection.



Time Aggregation: Based on the Time period field added to the model, you can choose from a range of Time Aggregation options to get the values displayed on the node accordingly.

Time AggregationDescription
1Sum of PeriodsAdds all of the period values
2AverageAverage of the period values gets displayed
3FormulaCalculates and applies the custom formula entered by the user
4LastOnly last period value of the node gets displayed
5CumulativeCumulative value of the period node gets displayed

The Normal node type presents the user with the Time Aggregation options as seen below.


Note
The constant node type doesn't have Time Aggregation as the value is fixed.

Although Sum of Periods and Cummulative both show the same result on top of the node, there is a difference between the two on the line chart. Click on the 'East' node to see a detailed report along with the graph plotted with the period values. From the below images we can see that the first one having Time Aggregation as Sum of Periods captures the movement of the period values and the latter one having Cummulative as Time Aggregation displays the running total.

'East' node having Time Aggregation as Sum of Periods


'East' node having Time Aggregation as Cumulative


Desired Trend: The Desired Trend can be set as Increase or Decrease depending on the context of the Node. For instance, income has an increasing trend and expenses have a decreasing trend.



The Desired trend impacts the Conditional Formatting while performing simulations. The impact of Desire Trend on Conditional Formatting has been discussed along with examples here.

Value Range: Value Range helps you to assign Minimum and Maximum value for a node. This option comes in handy at the time of simulation of the node for which threshold has been set to evaluate it's performance against the benchmark i.e. Constraint.



In the following example, Value Range has been added to a node and the Constraint for that particular node has been activated under Settings.

A. The Value Range for 'East' Node has been set as Min = 1 Million and Max = 10 Million. Notice that the Time Aggregation has been set as Sum of Periods and the number of periods for this node is 12. Therefore, the Value Range is taken as Min = 12 Million and Max = 120 Million.


B. After the Value Range has been set to the 'East' node, the same node is added to Constraint under the Navigation Panel. To add Constraint, Model -> Settings -> Navigation Panel -> Constraints and choose constraints from the drop-down to be listed.


C. The Constraint for 'East' node has been added with the Minimum and Maximum values as seen below. Notice that the node value is outside the Value Range and the node value is displayed in red color. The value gets displayed in white color if it lies within the range.


You can also learn about Constraints, here.

Dynamic Children: Dynamic Children is available to create a new node that is replicate of another existing node having Data Source as it's Calculation Method. The new node works like a Simple Model where you cannot configure the child nodes except for simulation. Dynamic Children Simulation has options to decide whether to allow simulation of child nodes. All allow simulation for all the child nodes, nodes w/o children allow simulation only for nodes without children i.e. nodes on the last level and none doesn't allow simulation for any of the nodes including the parent.



The below example shows how to create Dynamic Children from Data Source.

First, we create a new node 'Copy of Overall Result' with the Calculation method as Data Source and 'East' as Source Key. Here, we enable the option Dynamic Children from Data Source and choose 'None' as Dynamic Children Simulation.

As a result of the above configuration, you will see the screen as below image.


Now to be able to configure the child nodes of the node created using the option Dynamic Children from Data Source, you click on the magic wand available next to the top node. The entire ValQ model is now an Advanced Model as seen below.

#6. Conditional Formatting

Using the Conditional Formatting option, you will be able to apply the Alert Thresholds and Rules at the specific Node Level (see Figure below). For our example, the Node 'East' has been selected.


In the Conditional Formatting settings for the node "East", you have 3 different options as highlighted in the above Figure.

Global or Inherited Rule: When the option "Global or Inherited Rule" is selected, the Conditional Formatting Rules configured as Global Rule in the Settings tab will be applied to the node.


Custom Rule: Custom Rule option overwrites default or any inherited rule to the rule as per our preference. The Custom Rule option can be configured based on the variance percentage or value and it gets applied at the period level. Checking Apply to Descendants option applies the Custom Rule to the child nodes as well.


Custom Rule is indicated by the icon (see Figure below) on the node in the Navigation Panel.


Hidden Rule: With this option, you can observe that no range values can be provided for the Conditional Formatting and hence there will be no status color for the node upon being simulated.


In the below example, we choose the option for the Custom Rule as "Percentage" and also enable the property "Apply to Descendants". We set the threshold values as shown in the below Figure. After the simulation, you can find that the variance percentage value for the node "East" falls in the threshold range (5%)* which is between -10% to -1% and based on that condition, the status bar for the Node East is yellow color (see Figure below).




Also, you can observe that the Descendant node "Consumer" has been updated with the same threshold settings based on our configuration. The variance percentage value for the Descendant node "Consumer" falls in the threshold range (3%)* which is Above 1% and based on that condition, the status bar for the Descendant node "Consumer" is green color. While the node 'East' has a variance of (4%)* which is within the threshold range and hence the status bar is yellow. (see Figure below)



Note
"*" denotes that the metric values get calculated based on the Period Level. For example, when it is a Year, it will take the value of 12 being multiplied with the range value that we provide.

#7. Secondary KPIs

Secondary KPIs helps the user to aggregate and visualize additional or alternate KPI values in the node widget. Specifically, in a model with two data series, these are the set of secondary metrics captured under the primary value of the node

As a default, there are three secondary values displayed for a dual series of data in the following order

  1. Full Period Variance between baseline and comparison metric (in % and absolute values)
  2. Baseline metric value for the first period in the series
  3. Variance for the first period between baseline and comparison metric (in % and absolute values)
Secondary KPI option lets users customize these secondary values or metric according to their preference

Users can achieve this using:

I. Secondary KPI option based on Formulas
II. Secondary KPI option based on Template

I. Secondary KPI option based on Formulas

In the below model screenshot, the user can display the data value from Operating Profit %â and âOperating Expense Ratioâ as Secondary KPI values to the TOP node âOperating profitâ.


Under the 'Operating Profit' node setting's 'Secondary KPI' ribbon, you need to select the 'Formula' button and fill in the label name and the custom formula for his additional Secondary KPIs



Upon completing, the user can see âOperating Profit %â and âOps Expense Ratioâ values displayed as Secondary KPI values to âOperating profitâ


II. Secondary KPI option based on Template

In the model, the user can display âContributionâ of each of the child nodes Water, Beverages, and Others to their parent node Gross Profit. Users should define âContributionâ as a Formula in one of the nodes and call them out for âContributionâ calculation using the Template option at the other nodes.


In this example, the user can define the formula in 'Water' node under the node setting's Secondary KPI by selecting the 'Formula' button and defining the 'Custom Formula' Upon completion, 'Contribution %' is displayed as a Secondary value of the node widget.


To borrow this formula on other nodes, the user can fo into the Secondary KPI setting of the 'Beverages' node, select the 'Template' button, and choose to display the same formula as the 'Water' node. Users can instantly see the corresponding 'Contribution %' at the 'Beverage' node.


Similarly, following the same steps for the 'Other' node helps the user to instantly visualize âContribution %â for the child nodes to Gross Profit
